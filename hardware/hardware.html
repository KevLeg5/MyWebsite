Microcontroller Elevator |

Skills: MPLabs, Microcontroller Programming, Circuit Construction

Goal: Configure a microcontroller and a breadboard to mimic the behavior of an elevator moving between 9 separate floors fulfilling requests.

Process: Using an ATSAMD21E18A microcontroller along with various components (keypad, LCD, LEDs, and buzzer), we needed to develop a functional elevator system that could navigate floors 1-9 based on user input. The passenger would have to be able to choose a direction, a floor, and open/close the door of the elevator, all through the keypad inputs. The implementation required extensive microcontroller configuration, including port registers for input/output management, Timer/Counter (TC) registers for timing control and delay generation, system control registers for clock configuration, External Interrupt Controller (EIC) registers for keypad interrupt handling, and various LCD functionalities for digital and hexadecimal writing operations. I was primarily responsible for the wiring and the FSM-like code behind the elevator's logic.

Results: After writing ~570 lines of code and making numerous wiring adjustments, we successfully implemented and demonstrated a working elevator system that could respond to passenger floor selections and door controls through the keypad. We were the only group in this class to finish this assignment on time, and recieved a 100/100 on it.

CSE262 CPU |

Skills: VHDL, Quartus Prime, FPGA, Assembly

What is it? - A very rudamentary CPU to read basic assembly for a school lab.

Goal: The goal was to program an FPGA board in VHDL to be capable of reading specific 4-bit instructions from a premade ROM, and then perform ALU logic and update registers accordingly.

Process: This small CPU was an accumulation of previous labs, including 7-segment display LEDs and Finite State Machines. Our CPU consisted of two 4-bit data buses, an instruction register, operand register, accumulator register, Z-flag register to check equality, program counter, ALU, FSM (for flagging), and a 16 line ROM.
We designed and implemented four instructions: LD (load), ADDA (add), CMPA (compare), and BEQ (branch if equal). We then had to configure the ROM so that the CPU could start the accumulator out at 3 and counts by 3s until 9, then re-start the accumulator at 2 and count up by 3s, and do this forever.

Synopsys/Cadence Transistor Analysis |

Skills: Synopsys, Cadence, CMOS Transistors, MOSFET

What is it? - Deep analysis and design of CMOS inverters.

Goal: School labs intended to help understand the concepts of CMOS inverters, voltage thresholds, and high-low propogation.

Synopsys: This is a software designed to help us view graphical representations of .sw files, which contain information about specific CMOS transistors. Through analysis of these files, I was able to visualize the concepts of cutoff voltage and Shockley's Model.

Cadende: A tool for designining transistors manually for later development. We designed an inverter using this tool, with wells, active layers, metals, vias, and contacts.
